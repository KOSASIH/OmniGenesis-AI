## Getting Started

### Prerequisites

- Node.js >= 18.x
- Python >= 3.10
- Bun >= 1.3.x
- Git
- A desire to build something that has never existed before

### Fork & Clone

```bash
git clone https://github.com/YOUR_USERNAME/OmniGenesis-AI.git
cd OmniGenesis-AI
npm install
cd frontend && bun install && cd ..
```

### Branching Convention

```
feat/phase14-multiverse         New feature for Phase 14
feat/agi-temporal-reasoning     AGI cognitive module
fix/consciousness-waveform      Bug fix
docs/tokenomics-update          Documentation
test/singularity-bond           Tests for AGI contracts
refactor/swap-routing           Code refactor
```

## Pull Request Process

1. **Fork** the repository and create your branch from `main`
2. **Write tests** for any new smart contract functionality
3. **Ensure all tests pass**: `npx hardhat test`
4. **Check Syariah compliance** for any DeFi or financial features
5. **Update documentation** in `docs/` if your change affects architecture
6. **Write a clear PR description** explaining *what* and *why*
7. **Link related issues** in your PR description

### PR Title Format

```
feat(phase13/agi): add temporal reasoning improvements
fix(contracts/cscns): correct voidtime yield calculation
docs(readme): update Phase 14 roadmap
test(agi): add AGIGovernance emergency halt tests
```

## Code Standards

### Solidity
- Solidity `^0.8.24`
- OpenZeppelin contracts for standard patterns
- NatSpec comments on all public/external functions
- No magic numbers — use named constants
- Events for all state changes

### TypeScript / React
- TypeScript strict mode
- Framer Motion for all animations
- No inline styles — use Tailwind classes
- Components in `frontend/src/components/<domain>/`
- Dynamic imports for all panel components

### Python (Agents)
- Python 3.10+ type hints
- Docstrings on all agent classes
- Async-first architecture
